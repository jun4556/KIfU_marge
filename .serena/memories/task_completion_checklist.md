# タスク完了時のチェックリスト

## ビルド確認

### 1. コンパイルエラーチェック
変更したモジュールに応じてビルドを実行：

**apiモジュールを変更した場合:**
```cmd
cd api
ant clean
ant javac
```

**drawerモジュールを変更した場合:**
```cmd
cd drawer
ant clean
ant javac
```

### 2. GWTコンパイルチェック（drawerモジュール変更時）
```cmd
cd drawer
ant gwtc
```
- JavaScript変換が成功することを確認
- エラーが出ないか確認

### 3. 完全ビルドテスト
**apiモジュール:**
```cmd
cd api
ant dist
```

**drawerモジュール:**
```cmd
cd drawer
ant build
```

## コード品質チェック

### 1. コーディング規約遵守
- [ ] 命名規則に従っているか（camelCase, PascalCase）
- [ ] UTF-8エンコーディングで保存されているか
- [ ] 適切なパッケージ構造になっているか

### 2. GWT固有のチェック
- [ ] クライアント側コードは`client`パッケージにあるか
- [ ] サーバー側コードは`server`パッケージにあるか
- [ ] RPCサービスは同期・非同期インターフェースペアになっているか
- [ ] Serializable/IsSerializableを実装しているか（転送オブジェクト）

### 3. ドキュメント
- [ ] JavaDocコメントを追加したか（公開API）
- [ ] 複雑なロジックにコメントを追加したか
- [ ] 必要に応じてメモリファイルを更新したか

## テスト（手動）

### 1. 機能テスト
- [ ] 変更した機能が意図通り動作するか
- [ ] 既存機能に影響がないか（リグレッションテスト）

### 2. データベーステスト（DB変更時）
- [ ] SQLスクリプトが正しく実行されるか
- [ ] テーブル構造が期待通りか
- [ ] データの整合性が保たれているか

## Git操作（コミット前）

### 1. 変更確認
```cmd
git status
git diff
```

### 2. ステージング
```cmd
git add <changed-files>
```

### 3. コミット
```cmd
git commit -m "明確な変更内容の説明"
```

## デプロイ前チェック（本番環境へのデプロイ時）

### 1. WARファイル生成確認
```cmd
cd drawer
ant war
```

### 2. データベースマイグレーション
- [ ] 必要なSQLスクリプトを準備
- [ ] バックアップを取得済みか

### 3. 環境設定
- [ ] データベース接続情報が正しいか
- [ ] GWT SDKパスが正しいか（build.xml）

## 注意事項
- GWTコンパイルは時間がかかるため、小さな変更の場合はスーパーDevモードを使用することを検討
- データベーススキーマ変更時は、既存データとの互換性に注意
- 本番環境へのデプロイ前には必ずバックアップを取得
